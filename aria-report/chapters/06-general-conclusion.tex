\chapter*{General Conclusion}
\addcontentsline{toc}{chapter}{General Conclusion}

This internship transformed a real SOC pain point into a working, end-to-end solution. ARIA proves that heterogeneous security and observability telemetry --- host, network, runtime, and metric --- can be unified into one understandable, traceable, and actionable workflow.

\section*{Answer to the Initial Problem}
\addcontentsline{toc}{section}{Answer to the Initial Problem}

The initial problem asked how a SOC can reduce manual effort and response delay when alerts arrive from many disconnected tools, while keeping remediation controlled, auditable, and verifiable. ARIA answers this by connecting every stage of the response chain in one platform: collection, detection, normalization, filtering, deduplication, enrichment, correlation, AI-assisted explanation, controlled remediation, verification, and archiving. Raw technical events become normalized alerts, incidents, investigations, response proposals, execution records, and archived cases. Monitored VMs are prepared once, onboarded as scoped assets, and validated before any remediation is enabled.

\section*{Main Results}
\addcontentsline{toc}{section}{Main Results}

The central achievement is the balance between automation and analyst control. AI generates summaries, narratives, risk scores, and Ansible playbooks; the analyst approves, executes, and verifies. Every sensitive action passes validation, approval, admin-secret checks, execution tracking, and audit persistence. This human-in-the-loop design is essential in cybersecurity, where the wrong automated action can damage availability or security.

\section*{Difficulties Encountered}
\addcontentsline{toc}{section}{Difficulties Encountered}

\begin{itemize}[leftmargin=1cm]
\item \textbf{Tool and format diversity.} Wazuh, Suricata, Falco, Telegraf, Filebeat, and Elasticsearch do not share an operational language, so ARIA needed source-specific mappers and a common alert model.
\item \textbf{Alert noise.} Repeated and low-value alerts required filtering, deduplication, and severity handling.
\item \textbf{AI reliability.} LLM output can be incomplete or malformed, so parsing, fallbacks, validation, and human approval were necessary.
\item \textbf{Multi-server scoping.} Once several servers share one cluster, alerts, investigations, and remediation actions for one server must never be confused with another.
\end{itemize}

\section*{Future Work}
\addcontentsline{toc}{section}{Future Work}

The current project is a strong controlled-environment SOC/SOAR foundation. Main future improvements are:

\begin{itemize}[leftmargin=1cm]
\item \textbf{Broader monitoring-tool support:} extend ARIA's mappers and source configs to ingest alerts from additional EDR, SIEM, and cloud-security tools beyond Wazuh, Suricata, Falco, Filebeat, and Telegraf.
\item \textbf{Expanded asset scoping:} apply scoping consistently to all monitoring views that are currently global by design.
\item \textbf{Richer playbook libraries:} build reusable, tested playbooks for common incident classes.
\item \textbf{LLM guardrails:} improve prompt engineering, output evaluation, and safety checks.
\item \textbf{Reporting and integration:} add automated incident reports and broader enterprise log sources.
\item \textbf{Onboarding validation:} add automated tests for the monitored-VM bootstrap path.
\end{itemize}

\section*{Final Word}
\addcontentsline{toc}{section}{Final Word}

ARIA is more than a dashboard: it is a complete detection-to-response philosophy made concrete. By connecting monitoring, intelligence, and remediation in one controlled workflow, it shows that automation and human judgment can reinforce each other rather than compete. The platform improves visibility, reduces manual investigation effort, accelerates incident understanding, and provides a safer path from detection to response --- while keeping the analyst responsible for every critical decision. Built as a controlled-environment SOC/SOAR foundation, it is ready for internal use, extension, and the next generation of monitoring integrations.
